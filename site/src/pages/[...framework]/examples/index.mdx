## Framer Motion

NumberFlow was designed to integrate well with [Framer Motion layout animations](https://www.framer.com/motion/layout-animations/).

<FramerMotionLayoutAnimations client:visible>
<Code code={FramerMotionLayoutAnimationsCode} lang="tsx" />
</FramerMotionLayoutAnimations>

Note that we turned `NumberFlow` into a [`motion` component](https://www.framer.com/motion/component/#custom-components) so we could use the [`layout layoutRoot`](https://www.framer.com/motion/layout-animations/###item-is-performing-unwanted-layout-animations-as-surrounding-content-changes) props.
These are required whenever `NumberFlow` is used within a layout animation.

To synchronize NumberFlow's transitions with Framer Motion's layout animations,
the package exports a `useTiming` hook from `@number-flow/react/framer-motion` that
takes an `EffectTiming` object and converts it to a Framer transition.
Make sure that the provided `EffectTiming` is memoized or  defined outside of the component,
as shown above.
